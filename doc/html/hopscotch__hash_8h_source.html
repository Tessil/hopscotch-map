128 /*
129  * Each bucket may store up to three elements:
130  * - An aligned storage to store a value_type object with placement-new.
131  * - An (optional) hash of the value in the bucket.
132  * - An unsigned integer of type neighborhood_bitmap used to tell us which buckets in the neighborhood of the
133  * current bucket contain a value with a hash belonging to the current bucket.
134  *
135  * For a bucket 'b' a bit 'i' (counting from 0 and from the least significant bit to the most significant)
136  * set to 1 means that the bucket 'b+i' contains a value with a hash belonging to bucket 'b'.
137  * The bits used for that, start from the third least significant bit.
138  *
139  * The least significant bit is set to 1 if there is a value in the bucket storage.
140  * The second least significant bit is set to 1 if there is an overflow. More than NeighborhoodSize values
141  * give the same hash, all overflow values are stored in the m_overflow_elements list of the map.
142  */
143 static const size_t NB_RESERVED_BITS_IN_NEIGHBORHOOD = 2;
